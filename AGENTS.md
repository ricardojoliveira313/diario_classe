# Regras do Projeto — Diário de Classe

## Obrigatório: commit + push sempre
- Toda alteração no código DEVE ser commitada e pusheada imediatamente após ser feita.
- Não adianta fazer alteração local sem subir — o usuário precisa ver no ar.
- Sequência: `git add` → `git commit` → `git push` — sem exceção.

## Branch
- Trabalhar em `main` e fazer push direto para `origin/main`, salvo instrução contrária.
